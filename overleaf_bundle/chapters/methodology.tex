This chapter presents the methodology for analyzing predictive multiplicity through a
spatial lens. The framework is formulated independently of any particular dataset or
experimental configuration; its concrete instantiation is described in Chapter~4.

The methodological flow is as follows. We first construct a Rashomon set of near-optimal
models and compute observation-wise predictive variance as the primary multiplicity metric. We then build a feature-space graph to encode neighborhood relationships, quantify
global and local spatial clustering via Moran's $I$ and LISA, extract hotspot regions (HH
clusters) and their connected components, and finally compare the observed structure
against a permutation null. The resulting pipeline identifies \emph{where} in feature space
predictive instability concentrates and whether that concentration exceeds what would be
expected by chance.

\subsection{Problem Setting and Notation}

Let $(X, y)$ denote a supervised learning dataset with $N$ observations, where
$X \in \mathbb{R}^{N \times d}$ is the feature matrix and
$y \in \{0,1\}^N$ is a binary target variable. Let $\mathcal{F}$ denote a hypothesis class
of predictive models, potentially spanning multiple model families and hyperparameter
configurations.

We consider probabilistic classifiers $f \in \mathcal{F}$ that output predicted probabilities
$\hat{p}_{fi} \in [0,1]$ for each observation $i \in \{1,\dots,N\}$. Model performance is
measured by a loss function $L(f)$ evaluated on held-out data. Throughout this chapter,
the framework is formulated for binary probabilistic classification; extensions to other
prediction settings are discussed later in the thesis.

\subsection{Rashomon Set Construction}
\label{sec:meth-rashomon}

As discussed in Chapter~2, the classical Rashomon set is usually defined through a loss
tolerance around the best model \citep{breiman_statistical_2001, marx_predictive_2020,
semenova_existence_2022}. In this thesis, we use a finite, rank-based approximation
instead. This distinction is important: the empirical model set used here is not defined by
a fixed absolute or relative performance tolerance, but by selecting a fixed number of
near-optimal models from a trained candidate pool.

Let $\mathcal{F}$ denote the finite candidate pool of trained models. Each model
$f \in \mathcal{F}$ is evaluated on the validation set using a loss $L(f)$.
In this thesis, $L$ is the Brier score, because the subsequent multiplicity
analysis is based on predicted probabilities rather than only on hard class labels. Let
$f_{(1)}, \ldots, f_{(|\mathcal{F}|)}$ denote the models sorted by increasing validation
loss, so that
\[
L(f_{(1)}) \leq L(f_{(2)}) \leq \cdots
\leq L(f_{(|\mathcal{F}|)}).
\]
The empirical Rashomon approximation used in the main analyses is then
\[
\mathcal{R}_K = \{f_{(1)}, \ldots, f_{(K)}\}.
\]

The reason for using $\mathcal{R}_K$ rather than a fixed $\varepsilon$-based threshold is
practical comparability. A fixed performance tolerance can lead to very different numbers
of selected models across datasets, splits, and model families. This would make
observation-wise variance estimates, spatial hotspot analyses, and comparisons across
runs harder to interpret. By fixing $K$, each run is based on the same number of
near-optimal models, while still retaining the central idea of predictive multiplicity:
studying disagreement among models with very similar validation performance.

Unless stated otherwise, $\mathcal{R}_K$ refers to the global top-$K$ set selected from
the full candidate pool. For analyses that compare model families or study variation
within a family, we also use family-specific sets. If $\mathcal{F}_g \subset \mathcal{F}$
denotes the candidate models belonging to model family $g$, then
$\mathcal{R}^{(g)}_K$ denotes the top-$K$ models within that family according to
validation Brier score. This family-specific construction ensures that enough models from
each family are available for the attribution analyses.

\subsection{Observation-Wise Predictive Variance}
\label{sec:meth-metrics}

To quantify predictive multiplicity at the level of individual observations, we compute
predictive variance across the finite top-$K$ model set $\mathcal{R}_K$ defined in
Section~\ref{sec:meth-rashomon}. Predictive variance measures how much the predicted
probabilities for the same observation vary across near-optimal models. This follows the
score-based view of predictive multiplicity, where disagreement is not only measured
after converting probabilities into class labels, but also at the level of predicted scores or
probabilities \citep{watson-daniels_predictive_2023, hsu_rashomon_nodate}.

Let $K = |\mathcal{R}_K|$ and let $\hat{p}_{mi}$ denote the predicted probability for
observation $i$ from model $m \in \mathcal{R}_K$. The observation-wise predictive
variance is defined as
\[
v_i = \mathrm{Var}_{m \in \mathcal{R}_K}(\hat{p}_{mi}),
\quad i = 1,\dots,N.
\]
High values of $v_i$ indicate that near-optimal models assign substantially different
probabilities to observation $i$, while low values indicate agreement. Unlike purely
label-based disagreement measures, predictive variance preserves information about the
spread of predicted probabilities even when all models produce the same final class label.

An alternative score-based measure is \emph{Rashomon Capacity}
\citep{hsu_rashomon_nodate}. Like predictive variance, it is defined at the level of an
individual observation and captures variation in predicted scores across near-optimal
models. It uses an information-theoretic formulation, whereas predictive variance uses the
empirical spread of predicted probabilities in the finite model set. We use predictive
variance here because it is directly computable from $\mathcal{R}_K$, easy to interpret as
probability spread, and can be decomposed naturally in the later family- and
hyperparameter-level analyses.

A supplementary decision-level analysis based on the conflict ratio is reported in
Appendix~\ref{app:conflict-ratio}. This analysis is not part of the main methodological
pipeline, but serves as a robustness-style check for whether spatial structure is also visible
after thresholding predicted probabilities.

\subsection{Feature-Space Graph and Spatial Statistics}
\label{sec:meth-graph}

To study whether prediction instability is locally concentrated, the test observations are
represented as a graph in feature space. Each node in the graph is one test observation,
and edges connect observations that are similar according to the preprocessed input
features. Throughout this thesis, the word ``spatial'' therefore does not refer to geographic
space. It refers to neighborhood relationships in the feature space of the data.

The purpose of this graph is simple: if two observations are close in feature space, they
are treated as local neighbors. We can then ask whether observations with high predictive
variance tend to have neighbors that also have high predictive variance. The spatial
statistics in this thesis are implemented with PySAL; the notation and local-cluster labels
below follow the definitions used by PySAL's Moran and Local Moran implementations.

\paragraph{Graph construction.}
\label{sec:method-spatial}

For each dataset and run, the graph is constructed on the held-out test set, because this
is the set on which multiplicity is evaluated. We use the preprocessed test-set feature
vectors and compute Euclidean distances between observations. For each observation, its
$k$ nearest neighbors are identified, yielding an initial $k$-nearest-neighbor graph.

The nearest-neighbor relation is not necessarily symmetric: observation $j$ may be among
the $k$ nearest neighbors of observation $i$, even if $i$ is not among the $k$ nearest
neighbors of $j$. We therefore symmetrize the graph. An undirected edge is placed
between observations $i$ and $j$ if either observation is among the $k$ nearest neighbors
of the other. Let
\[
A \in \{0,1\}^{N \times N}
\]
denote the resulting symmetric adjacency matrix, where $A_{ij}=1$ means that
observations $i$ and $j$ are connected, and $A_{ij}=0$ otherwise.

The adjacency matrix is then row-standardized to obtain the spatial weight matrix $W$:
\[
W_{ij} =
\frac{A_{ij}}{\sum_{j'=1}^{N} A_{ij'}},
\quad i = 1,\dots,N.
\]
After this normalization, each row of $W$ sums to one. As a result, when we compute
$[W\mathbf{v}]_i$ (the spatial lag of $\mathbf{v}$ at observation $i$), we obtain the average value of $\mathbf{v}$ among the neighbors of
observation $i$, rather than the total sum of their values.

For any observation-level quantity $\mathbf{v}=(v_1,\dots,v_N)^\top$, the term
\[
[W\mathbf{v}]_i = \sum_{j=1}^{N} W_{ij} v_j
\]
is called the \emph{spatial lag} of $\mathbf{v}$ at observation $i$. In this thesis, it can be
understood as the average value of the quantity among the neighbors of observation $i$.
For example, if $v_i$ is predictive variance, then $[W\mathbf{v}]_i$ is the average
predictive variance in the local neighborhood of observation $i$.

\paragraph{Global Moran's $I$.}
\label{sec:meth-moran}

Global Moran's $I$ is a classical measure of spatial autocorrelation
\citep{moran1950notes}. It measures whether similar values tend to occur near each
other in a given neighborhood graph. In this thesis, it is used to test whether
high-variance observations tend to be close to other high-variance observations, and
low-variance observations tend to be close to other low-variance observations.

Let $\mathbf{v}=(v_1,\dots,v_N)^\top$ denote the observation-wise predictive variance
values on the test set, and let $\bar{v}$ be their mean. Because $W$ is row-standardized,
Moran's $I$ can be written as
\[
I =
\frac{\sum_{i=1}^N \sum_{j=1}^N W_{ij}(v_i-\bar{v})(v_j-\bar{v})}
{\sum_{i=1}^N (v_i-\bar{v})^2}.
\]
The numerator compares each observation's deviation from the mean with the deviations
of its neighbors. If observations with above-average variance tend to have neighbors that
also have above-average variance, and observations with below-average variance tend to
have below-average neighbors, the numerator becomes positive. In that case, Moran's
$I$ is positive, indicating spatial clustering. Values close to zero indicate that the
variance values are not systematically organized with respect to the graph. Negative
values would indicate that high values tend to be surrounded by low values, or vice versa.

As a simple example, suppose several similar observations all have high predictive
variance and are connected in the graph. This produces a positive contribution to Moran's
$I$, because both the observation and its neighbors are above the mean. If high-variance
observations were scattered randomly among low-variance observations, the global value
would be much closer to zero.

\paragraph{Local Moran statistics and LISA.}
\label{sec:meth-lisa}

Global Moran's $I$ summarizes the overall degree of spatial clustering, but it does not
show where the clustering occurs. For this reason, we also use Local Indicators of Spatial
Association (LISA), introduced by \citet{anselin1995local}. LISA decomposes spatial
association into observation-level statistics and identifies which observations contribute
to local clusters.

In the PySAL implementation, the input variable is standardized before the local Moran
statistics are computed. In this thesis, the input variable is the vector of predictive
variance values. Let
\[
z_i = \frac{v_i-\bar{v}}{s_v}
\]
denote the standardized predictive variance of observation $i$, where $\bar{v}$ is the mean
predictive variance across test observations and $s_v$ is the corresponding standard
deviation. The spatial lag of the standardized values is
\[
[W\mathbf{z}]_i = \sum_{j=1}^{N} W_{ij} z_j,
\]
that is, the weighted average of the standardized values among the neighbors of
observation $i$. The local Moran statistic used by PySAL is proportional to
\[
z_i [W\mathbf{z}]_i,
\]
up to a constant scaling factor. Therefore, its interpretation is straightforward: the
statistic is large and positive when observation $i$ and its neighbors are on the same side
of the mean, for example when both have above-average predictive variance.

To decide whether a local pattern is stronger than expected by chance, a permutation
test is used for each observation. The values of the neighboring observations are randomly
rearranged many times, and the observed local statistic is compared with the resulting
reference distribution. This yields one local $p$-value for each observation.

Because this produces many local tests at once, some observations may appear significant
by chance alone. We therefore apply the Benjamini--Hochberg procedure
\citep{benjamini_hochberg_1995}. This procedure controls the false discovery rate, that
is, the expected proportion of falsely selected observations among all observations declared
significant. It adjusts the set of local $p$-values so that the final LISA
labels are not based on uncorrected pointwise tests. Observations that remain significant
after this multiple-testing correction are assigned to one of the local spatial categories
below.

\paragraph{Hotspots, coldspots, and spatial outliers.}

Significant observations are classified according to the sign of their own standardized
value $z_i$ and the sign of the average standardized value in their neighborhood
$[W\mathbf{z}]_i$:

\begin{itemize}
  \item \textbf{High--High (HH):} $z_i > 0$ and $[W\mathbf{z}]_i > 0$. The observation has
  above-average predictive variance and is surrounded by neighbors with above-average
  predictive variance. These observations are called \emph{multiplicity hotspots}.

  \item \textbf{Low--Low (LL):} $z_i < 0$ and $[W\mathbf{z}]_i < 0$. The observation has
  below-average predictive variance and is surrounded by neighbors with below-average
  predictive variance. These observations are called \emph{coldspots}, because they mark
  locally stable regions.

  \item \textbf{High--Low (HL):} $z_i > 0$ and $[W\mathbf{z}]_i < 0$. The observation has
  above-average predictive variance, but its neighbors have below-average predictive
  variance. This corresponds to a high-variance spatial outlier.

  \item \textbf{Low--High (LH):} $z_i < 0$ and $[W\mathbf{z}]_i > 0$. The observation has
  below-average predictive variance, but its neighbors have above-average predictive
  variance. This corresponds to a low-variance observation inside a high-variance
  neighborhood.
\end{itemize}

Observations that are not significant after the Benjamini--Hochberg correction are labeled
\emph{non-significant}. Throughout the thesis, an \emph{HH region} refers to the set of
High--High observations, and an \emph{HH hotspot} refers to a local region of elevated
predictive variance.

\paragraph{Connected components of HH regions.}

The LISA classification identifies individual HH observations. To obtain larger coherent
regions, we restrict the symmetric kNN graph to the HH observations only and identify
its connected components. A connected component is a set of HH observations that are
linked to each other through paths in the graph. Each component is therefore interpreted
as a contiguous high-multiplicity region in feature space.

Very small components may be discarded to avoid treating isolated fragments as
substantive hotspot regions. The minimum component size used in the experiments is
specified in Chapter~4.


\subsection{Null Model for Significance Testing}
\label{sec:meth-null}

To ensure that detected spatial structure is not merely an artifact of the marginal
distribution of disagreement values, we use a null model based on permutation of the
prediction matrix itself. The null hypothesis is that the overall magnitude of model
disagreement is preserved, but its alignment with feature-space location is random.

The null operates on the prediction matrix $P \in \mathbb{R}^{K \times N}$. For each model
$m$ independently, the vector $(\hat{p}_{m1}, \dots, \hat{p}_{mN})$ is randomly permuted
across observations. This \emph{within-model permutation} preserves each model's
marginal distribution of predictions while destroying systematic alignment between
predictions and feature-space location. Any spatial clustering of the resulting predictive
variance values $v_i$ is therefore attributable to chance under this null model.

For each of $R$ permutations, observation-wise predictive variance and global Moran's
$I$ are recomputed. The empirical $p$-value is
\[
p = \frac{1 + \#\{\text{null} \geq \text{observed}\}}{R + 1}.
\]
This tests whether the observed spatial organization of predictive variance exceeds what
would arise from spatially unstructured predictions with the same model-wise marginal
prediction distributions.


\subsection{Methodological Assumptions}
\label{sec:meth-assumptions}

The proposed framework relies on several assumptions.

First, the feature-space representation induces a meaningful notion of local similarity,
such that nearby observations are comparable in terms of predictive behavior. The spatial
analysis therefore depends on the choice of feature transformation and distance metric.

Second, the finite candidate pool and the top-$K$ selection provide a practical
approximation of the Rashomon set of near-optimal models. The resulting multiplicity
estimates therefore depend on the model families and hyperparameter configurations
included in the candidate pool.

Third, the analysis is descriptive rather than causal. Identified hotspot regions indicate
where predictive multiplicity concentrates, but do not explain the underlying
data-generating mechanisms that give rise to disagreement between models.

Finally, the framework is defined for probabilistic classification models and relies on
their predicted probabilities. Extensions to other prediction settings (e.g., regression or
multiclass classification) require corresponding adaptations of the multiplicity metrics.

\subsection{Summary}

The methodology provides a unified framework for analyzing predictive multiplicity as a
spatial phenomenon. Its core steps are: (1) rank-based Rashomon approximation,
(2) observation-wise predictive variance, (3) feature-space kNN graph, (4) global
Moran's $I$ and local LISA for hotspot detection, (5) connected components of HH
regions, and (6) a prediction-permutation null for significance testing. Chapter~4
instantiates this framework on concrete datasets and experimental settings.
